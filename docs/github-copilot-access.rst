######################################################
LF Project Maintainer GitHub Copilot Enterprise Access
######################################################

Introduction
============

The Linux Foundation provides GitHub Copilot Enterprise access to maintainers of
LF open source projects that have elected into the starter tier (or better) of
Release Engineering support services. This gives maintainers full access to
GitHub Copilot features at an enterprise level, including code completion, chat,
and all available models. New models get enabled as they become available.

Eligibility
===========

To qualify for GitHub Copilot Enterprise access, you must meet the following
criteria:

- You hold a **verifiable maintainer role** in a public Linux Foundation project
  or repository.
- Your project has an active **starter tier (or better)** Release Engineering
  support contract.
- Your project repositories must **exist on GitHub**.

Requesting Access
=================

To request GitHub Copilot Enterprise access, file a support ticket through the
LF Helpdesk:

.. note::

   Use the following temporary ticket link while the team finalizes the
   permanent request type:

   https://jira.linuxfoundation.org/plugins/servlet/desk/portal/2/create/1275

Include the following information in your ticket:

#. **GitHub username** — your GitHub account that will receive Copilot access.
#. **Top-level project** — the Linux Foundation top-level project you are part
   of (e.g., ONAP, OpenDaylight, Hyperledger).
#. **Specific project** — the specific project or repository within that
   top-level project where you hold a maintainer role.
#. **Proof of maintainer role** — evidence that you are a maintainer (e.g., a
   link to an INFO.yaml, MAINTAINERS, or CODEOWNERS file, a TSC resolution, or
   other verifiable record).

What Happens Next
=================

#. A **support engineer** reviews your ticket and verifies the information
   provided.
#. Upon approval, you will receive an **invitation to the
   linuxfoundation-maintainers GitHub organization**.
#. Once you accept the invitation, you will have access to all Copilot models at
   an enterprise level.

Verifying Copilot Billing
-------------------------

If you have access to GitHub Copilot through other organizations or
enterprises, verify that your Copilot billing points to the right entity:

#. Navigate to https://github.com/settings/copilot/features
#. Confirm that the billing shows **Linux Foundation OSS Maintainers**.

This ensures you use the LF-provided Copilot Enterprise access rather than
a personal or other organizational subscription.

Maintaining Access
==================

To keep your GitHub Copilot Enterprise access:

- You must remain an **active maintainer** of at least one LF open source
  project that has a starter tier (or better) support contract.
- Access undergoes **re-evaluation every 3 months**. If you no longer meet the
  eligibility criteria at the time of review, the team may remove your access.
- Any evidence of **misuse** of the access will be grounds for revocation.
